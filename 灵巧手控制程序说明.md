# 灵巧手控制程序说明

本文档说明 `3D_Hand_2` 固件实现了什么功能、硬件应该怎么接、串口命令怎么发、如何编译烧录，以及调试时应该检查哪些问题。

## 1. 程序实现的功能

这个工程是基于 STM32CubeMX 生成的 STM32F103C8Tx CMake 工程，在生成代码基础上加入了五指灵巧手控制逻辑。程序通过串口接收上位机命令，再输出 5 路 50 Hz PWM 控制 5 个 SG90 舵机。

整体控制链路如下：

```text
上位机 / ROS / 串口助手
  -> USART1 文本命令
  -> STM32F103C8Tx
  -> TIM2/TIM3 输出 5 路 PWM
  -> 5 个 SG90 舵机
  -> 灵巧手张开、闭合或单指动作
```

当前固件支持这些能力：

- 五指整体张开、整体闭合。
- 使用 0.0 到 1.0 的比例值控制整体抓握程度。
- 单独控制 5 根手指的闭合比例。
- 串口心跳测试。
- 串口查询当前 PWM 状态。
- 命令超时保护：超过 500 ms 没收到新控制命令时，目标位置自动回到张开状态。
- 平滑运动：舵机不会一次跳到目标位置，而是每 20 ms 最多改变 10 us PWM 脉宽。
- PWM 脉宽保护：最终写入比较寄存器前会限制在 500 us 到 2500 us 之间。

## 2. 硬件连接

### 2.1 MCU 和串口

目标芯片：

```text
STM32F103C8Tx
```

串口使用 USART1：

```text
USART1_TX: PA9
USART1_RX: PA10
波特率:    115200
格式:      8N1
```

如果你使用 USB-TTL 模块连接电脑，注意 TX/RX 需要交叉：

```text
USB-TTL TX -> STM32 PA10
USB-TTL RX -> STM32 PA9
USB-TTL GND -> STM32 GND
```

### 2.2 五路舵机 PWM

当前程序里的舵机编号和引脚映射如下：

| 舵机编号 | 手指 | STM32 引脚 | 定时器通道 |
| --- | --- | --- | --- |
| 0 | 拇指 | PA0 | TIM2_CH1 |
| 1 | 食指 | PA1 | TIM2_CH2 |
| 2 | 中指 | PA2 | TIM2_CH3 |
| 3 | 无名指 | PA3 | TIM2_CH4 |
| 4 | 小指 | PA6 | TIM3_CH1 |

TIM2 和 TIM3 的 PWM 配置：

```text
定时器输入时钟: 72 MHz
Prescaler:      71
计数频率:       1 MHz
Period:         19999
PWM 周期:       20000 us = 20 ms = 50 Hz
初始 Pulse:     1000 us
```

### 2.3 舵机供电

SG90 舵机不能直接接 24 V，也不建议从 STM32 板子的 5 V 引脚取电带多个舵机。推荐供电方式：

```text
24 V 电源
  -> DC-DC 降压模块
  -> 5 V 或 6 V
  -> SG90 舵机 VCC
```

必须共地：

```text
STM32 GND
USB-TTL GND
DC-DC GND
舵机电源 GND
```

如果不共地，PWM 信号没有共同参考电平，舵机可能不动作、乱动或抖动。

## 3. 启动后的行为

上电后程序会依次执行：

1. 初始化 HAL 和系统时钟。
2. 配置 USART1。
3. 配置 TIM2、TIM3 的 PWM 输出。
4. 将 5 个舵机的当前位置和目标位置设为张开位置。
5. 开始 USART1 中断接收。
6. 通过串口发送：

```text
3D_HAND_READY
```

如果串口助手打开正确，复位 STM32 后应能看到 `3D_HAND_READY`。

## 4. 串口命令格式

所有命令都是 ASCII 文本命令，必须以换行结束。

推荐使用：

```text
\n
```

或者：

```text
\r\n
```

程序会忽略 `\r`，用 `\n` 作为一条命令结束标志。如果串口助手只发送文本但不发送换行，STM32 会一直等待，不会执行命令。

### 4.1 PING 心跳测试

发送：

```text
PING
```

返回：

```text
PONG
```

这个命令用于确认串口收发正常。

### 4.2 张开

发送：

```text
OPEN
```

或者：

```text
O
```

返回：

```text
OK,OPEN
```

效果：5 根手指目标位置全部设为张开位置。

### 4.3 闭合

发送：

```text
CLOSE
```

或者：

```text
C
```

返回：

```text
OK,CLOSE
```

效果：5 根手指目标位置全部设为闭合位置。

### 4.4 整体抓握比例

发送格式：

```text
G,value
```

`value` 是抓握比例：

```text
0.0 表示完全张开
1.0 表示完全闭合
0.5 表示半闭合
```

示例：

```text
G,0.000
G,0.300
G,0.500
G,1.000
```

正常返回：

```text
OK,G
```

格式错误时返回：

```text
ERR,G
```

注意：解析函数会把大于 1.0 的数限制到 1.0，例如 `G,2.0` 最终会按完全闭合处理。不支持负数，不要发送 `G,-0.5`。

### 4.5 单独控制五根手指

发送格式：

```text
F,thumb,index,middle,ring,pinky
```

五个参数分别对应：

```text
拇指,食指,中指,无名指,小指
```

每个参数都是 0.0 到 1.0 的比例。

示例：

```text
F,0.0,1.0,1.0,1.0,0.5
```

含义：

```text
拇指张开
食指闭合
中指闭合
无名指闭合
小指半闭合
```

正常返回：

```text
OK,F
```

如果 5 个值后面还有多余字符，会返回：

```text
ERR,F
```

注意：当前代码对部分不完整的 `F` 命令会直接忽略，例如缺少某个值或缺少逗号时可能没有返回。因此调试时建议先严格按完整格式发送。

### 4.6 查询状态

发送：

```text
STATUS
```

或者：

```text
S
```

返回格式：

```text
STATUS,c0,c1,c2,c3,c4,t0,t1,t2,t3,t4
```

含义：

```text
c0~c4: 5 个舵机当前 PWM 脉宽，单位 us
t0~t4: 5 个舵机目标 PWM 脉宽，单位 us
```

示例返回：

```text
STATUS,1000,1000,1000,1000,1000,1900,2000,2000,1950,1900
```

这个例子表示当前还在张开位置，但目标已经是闭合位置，程序会继续平滑移动。

### 4.7 未知命令

如果发送无法识别的命令，例如：

```text
HELLO
```

返回：

```text
ERR,UNKNOWN
```

## 5. 舵机行程和控制参数

舵机张开/闭合参数在 [Core/Src/main.c](Core/Src/main.c) 中：

```c
static const uint16_t servo_open_us[HAND_SERVO_COUNT] = {
  1000U, 1000U, 1000U, 1000U, 1000U
};

static const uint16_t servo_close_us[HAND_SERVO_COUNT] = {
  1900U, 2000U, 2000U, 1950U, 1900U
};
```

默认行程：

| 手指 | 张开 PWM | 闭合 PWM |
| --- | ---: | ---: |
| 拇指 | 1000 us | 1900 us |
| 食指 | 1000 us | 2000 us |
| 中指 | 1000 us | 2000 us |
| 无名指 | 1000 us | 1950 us |
| 小指 | 1000 us | 1900 us |

整体抓握命令 `G,value` 会在张开值和闭合值之间线性插值。例如拇指：

```text
G,0.0 -> 1000 us
G,0.5 -> 1450 us
G,1.0 -> 1900 us
```

单指命令 `F,...` 对每根手指单独做同样的插值。

## 6. 平滑运动和安全保护

### 6.1 平滑运动

主循环每 20 ms 调用一次 `Hand_Tick()`。每次 tick 中，每个舵机最多移动 10 us：

```text
控制周期: 20 ms
最大步进: 10 us / 20 ms
```

这样做的目的：

- 减小 SG90 瞬间冲击。
- 降低连杆卡住时的机械冲击。
- 让动作更平滑。

### 6.2 命令超时保护

如果超过 500 ms 没收到新的控制命令，程序会自动把目标位置设为张开：

```text
HAND_COMMAND_TIMEOUT_MS = 500
```

这意味着：

- 如果上位机持续控制，需要周期性发送命令，频率建议高于 2 Hz。
- 如果只是手动测试，发送 `CLOSE` 后 500 ms 没有新命令，目标会自动回到张开。
- 如果想保持闭合，可以周期性发送 `CLOSE` 或 `G,1.0`。

### 6.3 PWM 限幅

实际写入 PWM 前，程序会把脉宽限制在：

```text
500 us 到 2500 us
```

这是最后一道软件保护，避免异常参数直接写出过大的 PWM。

## 7. 推荐调试流程

第一次接线和上电时建议按这个顺序调试。

### 7.1 不接舵机，只测 STM32 和串口

1. 给 STM32 上电。
2. 打开串口助手。
3. 设置 `115200 8N1`。
4. 复位 STM32。
5. 确认收到：

```text
3D_HAND_READY
```

发送：

```text
PING
```

确认收到：

```text
PONG
```

### 7.2 接一个舵机测试 PWM

先只接一个舵机，例如拇指 PA0。

发送：

```text
OPEN
```

再发送：

```text
G,0.2
```

确认动作方向和机械结构是否安全。不要一开始就发送 `CLOSE` 或 `G,1.0`，避免连杆顶死。

### 7.3 逐步增加闭合行程

如果机械结构还没有标定，建议先把 `servo_close_us` 改小，例如：

```text
1500 us
```

确认不会顶死、不会持续大电流、舵机不会明显发热后，再逐步增加：

```text
1600 us
1700 us
1800 us
1900 us
2000 us
```

每根手指都应该单独标定，不要假设所有舵机和连杆行程完全一致。

### 7.4 接入全部舵机

全部舵机接入后，先测试：

```text
OPEN
G,0.2
G,0.4
G,0.6
```

最后再测试：

```text
G,1.0
```

如果出现某根手指先顶死，应降低对应手指的 `servo_close_us`。

## 10. CubeMX 配置说明

当前 `.ioc` 文件已经配置为和代码一致：

- TIM2: CH1、CH2、CH3、CH4 PWM。
- TIM3: CH1 PWM。
- USART1: PA9/PA10，115200。
- USART1 中断开启，优先级 1。
- HSE: External Clock Source，对应代码里的 `RCC_HSE_BYPASS`。

注意：如果你使用的是常见带 8 MHz 晶振的 STM32F103C8T6 小板，而不是外部有源时钟输入，`RCC_HSE_BYPASS` 可能不适合。现象通常是程序起不来、串口没有 `3D_HAND_READY`。这种情况下需要把 HSE 改成晶振模式，并把代码里的：

```c
RCC_OscInitStruct.HSEState = RCC_HSE_BYPASS;
```

改为：

```c
RCC_OscInitStruct.HSEState = RCC_HSE_ON;
```

修改时钟后需要重新生成或重新编译。

## 11. 重新用 CubeMX 生成代码时的注意事项

当前 `main.c` 里 USART1、TIM2、TIM3 初始化函数是在 `USER CODE` 区域手写的。现在 `.ioc` 也配置了这些外设，所以如果你重新点击 CubeMX 的 `Generate Code`，可能会出现两种情况：

- CubeMX 保留现有 `USER CODE`，工程仍然可用。
- CubeMX 又生成一份同名的 `MX_USART1_UART_Init()`、`MX_TIM2_Init()`、`MX_TIM3_Init()` 或外设句柄，导致重复定义。

重新生成后如果编译报重复定义，重点检查：

```text
Core/Src/main.c
Core/Src/usart.c
Core/Src/tim.c
Core/Inc/usart.h
Core/Inc/tim.h
```

当前工程还需要 CMake 列表包含这些 HAL 源文件：

```text
Drivers/STM32F1xx_HAL_Driver/Src/stm32f1xx_hal_tim.c
Drivers/STM32F1xx_HAL_Driver/Src/stm32f1xx_hal_tim_ex.c
Drivers/STM32F1xx_HAL_Driver/Src/stm32f1xx_hal_uart.c
```

并确认 [Core/Inc/stm32f1xx_hal_conf.h](Core/Inc/stm32f1xx_hal_conf.h) 里启用了：

```c
#define HAL_TIM_MODULE_ENABLED
#define HAL_UART_MODULE_ENABLED
```

## 12. 常见问题排查

### 12.1 串口没有任何输出

检查：

- 波特率是否是 115200。
- 串口助手是否打开了正确 COM 口。
- TX/RX 是否交叉连接。
- USB-TTL 和 STM32 是否共地。
- STM32 是否真正复位运行。
- HSE 模式是否和板子硬件一致。

### 12.2 能收到 `3D_HAND_READY`，但发命令没有反应

检查：

- 串口助手是否发送换行 `\n` 或 `\r\n`。
- 命令是否是大写。
- 命令中是否有空格。当前解析器不处理空格，例如不要发送 `G, 0.5`。
- 先发送 `PING`，确认能返回 `PONG`。

### 12.3 串口有返回，但舵机不动

检查：

- 舵机电源是否是 5 V 或 6 V。
- 舵机电源和 STM32 是否共地。
- 舵机信号线是否接到正确 PWM 引脚。
- PWM 引脚是否被其他外设占用。
- 舵机本身是否损坏。

### 12.4 舵机抖动或复位

常见原因：

- 舵机电源电流不够。
- 多个 SG90 同时动作造成电压跌落。
- 没有共地。
- 闭合行程过大，机械结构顶死。

处理建议：

- 使用更大电流的 DC-DC 模块。
- 降低 `servo_close_us`。
- 先单个舵机调试，再接入全部舵机。
- 上电后先发送 `OPEN`，确认初始姿态安全。

### 12.5 发送 `CLOSE` 后马上又张开

这是当前程序的超时保护导致的。`CLOSE` 只更新一次目标位置，如果 500 ms 内没有新命令，程序会自动把目标位置设回张开。

如果需要持续保持闭合，上位机应周期性发送：

```text
CLOSE
```

或者：

```text
G,1.0
```

发送周期建议小于 500 ms，例如每 100 ms 发送一次。

## 13. 最小使用示例

串口助手设置：

```text
波特率: 115200
数据位: 8
停止位: 1
校验:   None
换行:   发送 LF 或 CRLF
```

测试顺序：

```text
PING
OPEN
G,0.2
G,0.4
G,0.6
STATUS
G,0.0
```

如果这些命令都有正常返回，并且舵机按预期动作，就可以接入 ROS 或其他上位机程序，让上位机周期性发送 `G,value` 或 `F,...` 命令控制灵巧手。
