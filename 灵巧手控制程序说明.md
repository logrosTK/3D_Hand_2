# 灵巧手 STM32 控制程序说明

这个工程是你用 STM32CubeMX 生成的 `3D_Hand_2`，我在它的基础上加入了 SG90 灵巧手控制功能。

控制链路：

```text
ROS 端 / VR 扳机
  ↓
串口发送 G,0.000 ~ G,1.000
  ↓
STM32 USART1 接收
  ↓
TIM2/TIM3 输出 5 路 50Hz PWM
  ↓
控制 5 个 SG90 舵机
```

## 1. 默认硬件连接

串口：

```text
USART1_TX: PA9
USART1_RX: PA10
波特率: 115200
格式: 8N1
```

舵机 PWM：

```text
拇指:   PA0  TIM2_CH1
食指:   PA1  TIM2_CH2
中指:   PA2  TIM2_CH3
无名指: PA3  TIM2_CH4
小指:   PA6  TIM3_CH1
```

SG90 不能直接接比赛的 24V 电源。正确接法：

```text
24V XT30
  ↓
DC-DC 降压到 5V 或 6V
  ↓
SG90 舵机 VCC
```

必须共地：

```text
STM32 GND
舵机电源 GND
降压模块 GND
```

## 2. 串口命令

支持这些命令，每条命令以换行结尾：

```text
G,0.000
G,0.500
G,1.000
OPEN
CLOSE
PING
F,0.0,1.0,1.0,1.0,0.5
```

最常用的是 `G`：

```text
G,0.000  五指张开
G,1.000  五指闭合
```

`F` 可以单独控制五根手指：

```text
F,拇指,食指,中指,无名指,小指
```

例子：

```text
F,0.0,1.0,1.0,1.0,0.5
```

表示拇指张开、食指中指无名指闭合、小指半闭合。

## 3. 安全逻辑

如果 STM32 超过 500ms 没收到新命令，会自动把目标位置设为张开，避免失控夹紧。

舵机不是一步跳到目标角度，而是每 20ms 最多变化 10us，动作会比较平滑，也能减少 SG90 卡顿。

## 4. 舵机行程标定

行程参数在：

```text
Core/Src/main.c
```

搜索：

```c
servo_open_us
servo_close_us
```

当前默认：

```text
张开: 1000us
闭合: 拇指1900us，食指2000us，中指2000us，无名指1950us，小指1900us
```

第一次上电不要直接装上全部连杆猛测。建议先把闭合值改小，例如：

```text
1500us
```

确认不顶死、不发热、不持续嗡嗡响，再慢慢加到 1800us、1900us、2000us。

## 6. 重新用 CubeMX 生成代码时注意

你现在的 `.ioc` 里没有配置 USART1、TIM2、TIM3，这些外设是我在 USER CODE 里手动初始化的。

如果以后你重新 Generate Code，通常 USER CODE 区会保留，但这个文件可能会被覆盖：

```text
cmake/stm32cubemx/CMakeLists.txt
```

如果编译时又提示找不到 TIM/UART，把这些 HAL 文件加回来：

```text
Drivers/STM32F1xx_HAL_Driver/Src/stm32f1xx_hal_tim.c
Drivers/STM32F1xx_HAL_Driver/Src/stm32f1xx_hal_tim_ex.c
Drivers/STM32F1xx_HAL_Driver/Src/stm32f1xx_hal_uart.c
```

并确认 `Core/Inc/stm32f1xx_hal_conf.h` 里有：

```c
#define HAL_TIM_MODULE_ENABLED
#define HAL_UART_MODULE_ENABLED
```
